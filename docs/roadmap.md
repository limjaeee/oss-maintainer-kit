# Roadmap

## 0.1

- Local CLI for PR review, issue triage, and release notes.
- JSON output for GitHub Actions and agent workflows.
- Tested core analysis and prompt generation.

## 0.2

- GitHub Action comments for pull requests.
- Repository maintainer profile file, such as `.maintainer-kit.yml`.
- OpenAI Responses API adapter behind explicit opt-in.

## 0.3

- Security-sensitive change checklist.
- Release candidate workflow with migration-note detection.
- Maintainer feedback loop for false positives and project-specific conventions.

## Principles

- Assist maintainers; do not replace maintainer judgment.
- Make uncertainty visible.
- Prefer concrete next actions over generic summaries.
- Keep automation auditable in public repositories.
